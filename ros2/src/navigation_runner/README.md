## 一、功能包概述

### 1.1 核心功能与设计目标

`navigation_runner` 是 NavRL 项目的**导航执行模块**，作为连接**仿真训练**与**真实部署**的桥梁，实现基于强化学习策略的实时自主导航。其核心功能包括：

| 功能模块 | 说明 |
|---------|------|
| **RL 策略推理** | 加载训练好的 PPO 策略模型，实时推理导航动作 |
| **多传感器融合** | 集成 LiDAR 点云、动态障碍物检测、里程计信息 |
| **安全动作生成** | 基于 ORCA 算法的安全速度修正 |
| **目标坐标系控制** | 旋转不变性的导航策略执行 |
| **航向控制** | PID 控制器实现精准航向调整 |

## 三、核心算法详解

### 3.1 PPO 策略网络

**网络架构**：

```
观测输入
    │
    ├──► LiDAR [1, 36, 4] ──► CNN ──┐
    │                                 │
    ├──► State [8] ──────────────────┼──► Concat ──► MLP [256, 256] ──► Actor/Critic
    │                                 │
    └──► Dynamic Obstacle [5, 10] ──► MLP ──┘

Actor (策略网络):
    MLP ──► Alpha, Beta ──► Beta Distribution ──► Action

Critic (价值网络):
    MLP ──► State Value
```

**关键特性**：
- **Beta 分布策略**：动作输出范围 [0, 1]，适合连续控制
- **目标坐标系**：状态和动作在目标坐标系下表示，实现旋转不变性
- **特征融合**：CNN (LiDAR) + MLP (状态) + MLP (动态障碍物)

### 3.2 ORCA 安全动作算法

**Optimal Reciprocal Collision Avoidance (ORCA)** 是一种分布式避障算法。

**核心原理**：

```
┌─────────────────────────────────────────────────────────────────┐
│                     ORCA 算法流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 速度障碍 (Velocity Obstacle) 计算                           │
│                                                                 │
│     对于每个障碍物，计算会导致碰撞的速度集合                      │
│     VO = {v | ∃t>0, (p + v·t) ∩ (p_obs + v_obs·t) ≠ ∅}         │
│                                                                 │
│  2. ORCA 平面生成                                               │
│                                                                 │
│     在当前速度 v 和 VO 边界之间构建约束平面                      │
│     plane: n·(v' - v) ≥ 0                                       │
│                                                                 │
│  3. 线性规划求解                                                │
│                                                                 │
│     在满足所有 ORCA 约束的条件下，                 │
│     找到最接近 RL 建议速度 v_rl 的安全速度 v_safe               │
│                                                                 │
│     min ||v_safe - v_rl||²                                      │
│     s.t. 所有 ORCA 约束                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```


### 3.3 坐标变换系统

**目标坐标系变换**是实现旋转不变性的关键：

```
世界坐标系 (World Frame)
    │
    ├──► 目标方向向量 target_dir = goal - current_pos
    │
    ▼
目标坐标系 (Goal Frame)
    • X轴: 指向目标方向（水平）
    • Y轴: 垂直于X轴（水平）
    • Z轴: 垂直向上
    
    变换矩阵 R = [target_dir_norm, perp_dir, z_axis]
    
    v_goal = R^T · v_world  (世界→目标)
    v_world = R · v_goal    (目标→世界)
```

**优势**：
- 策略网络无需学习绝对方向
- 同一策略适用于任意起始-目标组合
- 提高泛化能力

## 四、ROS2 接口详解

### 4.1 订阅的话题 (Subscribers)

| 话题名 | 消息类型 | 说明 | 来源 |
|--------|----------|------|------|
| `/unitree_go2/odom` | `nav_msgs/Odometry` | 机器人里程计 | 飞控/定位系统 |
| `/goal_pose` | `geometry_msgs/PoseStamped` | 目标位置 | 用户/规划器 |
| `/navigation_emergency_stop` | `std_msgs/Bool` | 紧急停止信号 | 安全监控 |

### 4.2 发布的话题 (Publishers)

| 话题名 | 消息类型 | 说明 |
|--------|----------|------|
| `/unitree_go2/cmd_vel` | `geometry_msgs/Twist` | 速度控制指令 |
| `/navigation_runner/goal` | `visualization_msgs/MarkerArray` | 目标可视化 |

### 4.3 服务客户端 (Service Clients)

| 服务名 | 服务类型 | 说明 | 提供方 |
|--------|----------|------|--------|
| `/occupancy_map/raycast` | `map_manager/srv/RayCast` | LiDAR 射线检测 | map_manager |
| `/onboard_detector/get_dynamic_obstacles` | `onboard_detector/srv/GetDynamicObstacles` | 动态障碍物查询 | onboard_detector |
| `/safe_action/get_safe_action` | `navigation_runner/srv/GetSafeAction` | 安全动作计算 | SafeAction Node |

### 4.4 服务服务器 (Service Servers)

| 服务名 | 服务类型 | 说明 |
|--------|----------|------|
| `/safe_action/get_safe_action` | `navigation_runner/srv/GetSafeAction` | 安全动作生成 |

**服务定义** (`GetSafeAction.srv`)：

```srv
# Request
geometry_msgs/Point agent_position      # 机器人当前位置
geometry_msgs/Vector3 agent_velocity    # 机器人当前速度
float64 agent_size                      # 机器人半径

# 动态障碍物信息
geometry_msgs/Vector3[] obs_position    # 障碍物位置列表
geometry_msgs/Vector3[] obs_velocity    # 障碍物速度列表
geometry_msgs/Vector3[] obs_size        # 障碍物尺寸列表

# 静态障碍物信息 (LiDAR点云)
float64[] laser_points                  # LiDAR 点云
float64 laser_range                     # LiDAR 探测范围
float64 laser_res                       # LiDAR 分辨率

float64 max_velocity                    # 最大速度限制
geometry_msgs/Vector3 rl_velocity       # RL 建议速度
---
# Response
geometry_msgs/Vector3 safe_action       # 安全动作速度
```

## 七、控制流程详解

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         导航控制主循环                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   初始化                                                                    │
│     ├── 加载 PPO 策略模型 (navrl_checkpoint.pt)                            │
│     ├── 初始化 PID 控制器                                                   │
│     └── 等待传感器服务就绪                                                   │
│         │                                                                   │
│         ▼                                                                   │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │                      定时回调 (20Hz)                                 │  │
│   │                                                                     │  │
│   │  1. 获取传感器数据                                                   │  │
│   │     ├── 调用 RayCast 服务获取 LiDAR 点云                            │  │
│   │     └── 调用 GetDynamicObstacles 获取动态障碍物                      │  │
│   │                                                                     │  │
│   │  2. 航向控制                                                         │  │
│   │     ├── 计算当前航向与目标方向夹角                                     │  │
│   │     ├── 如果角度差 > 0.3 rad                                          │  │
│   │     │   └── 仅发布角速度指令（原地旋转对准目标）                        │  │
│   │     └── 否则继续下一步                                                │  │
│   │                                                                     │  │
│   │  3. 策略推理                                                         │  │
│   │     ├── 构建观测张量 (LiDAR + State + Dynamic Obstacle)              │  │
│   │     ├── 转换到目标坐标系                                              │  │
│   │     ├── PPO 网络前向传播                                              │  │
│   │     │   └── 输出归一化动作 [0, 1]                                     │  │
│   │     ├── 反归一化到速度范围 [-vel_limit, vel_limit]                    │  │
│   │     └── 转换回世界坐标系                                              │  │
│   │                                                                     │  │
│   │  4. 安全检查                                                         │  │
│   │     ├── 检查是否有障碍物在范围内                                       │  │
│   │     │   ├── 无: 直接朝目标方向运动（简化模式）                          │  │
│   │     │   └── 有: 调用 SafeAction 服务                                   │  │
│   │     │       └── ORCA 算法修正速度                                      │  │
│   │     └── 检查紧急停止信号                                               │  │
│   │                                                                     │  │
│   │  5. 发布控制指令                                                      │  │
│   │     └── Twist 消息 (linear.x, linear.y, linear.z, angular.z)          │  │
│   │                                                                     │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 八、技术亮点

| 特点 | 说明 |
|------|------|
| **Sim-to-Real 迁移** | 训练于 Isaac Sim，直接部署到真实机器人 |
| **旋转不变性** | 目标坐标系设计，策略与绝对方向无关 |
| **安全保证** | ORCA 算法确保与障碍物无碰撞 |
| **实时性能** | 20Hz 控制频率，满足实时性要求 |
| **模块化设计** | 感知、规划、控制分离，易于维护 |
| **多传感器融合** | LiDAR + 动态障碍物检测 + 里程计 |
| **混合控制** | RL 策略 + 传统 PID + 安全约束 |

`navigation_runner` 是 NavRL 项目的核心执行模块，成功将仿真训练的 RL 策略部署到真实机器人平台，实现了安全、高效的自主导航。
